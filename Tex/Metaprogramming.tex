    \subsection{Why Metaprogramming?}\label{why-metaprogramming}

\begin{itemize}
\itemsep1pt\parskip0pt\parsep0pt
\item
  Metaprogramming is used to write code at parsing time in order to help
  the compiler achieve good performance
\item
  Metaprogramming is used to write Domain-Specific Languages (DSLs) to
  make easy syntax for complicated functions
\end{itemize}

    \subsection{Examples of Improving
Performance}\label{examples-of-improving-performance}

\begin{itemize}
\itemsep1pt\parskip0pt\parsep0pt
\item
  \texttt{@fastmath 3*5}
\item
  ParallelAccelerator's \texttt{@acc}
\item
  \texttt{@simd}
\item
  \texttt{@parallel}
\item
  \texttt{Threads.@threads}
\item
  Even DSLs will use symbolic mathematics to automatically simplify a
  problem for you!
\end{itemize}

    \subsection{Writing Your Own Macros}\label{writing-your-own-macros}

An easy way to write macros is the ``QuoteLater'' method:

\begin{itemize}
\itemsep1pt\parskip0pt\parsep0pt
\item
  Write a macro, make it work (in the REPL)
\item
  Later, quote it and interpolate in the value
\end{itemize}

The Julia manual is very complete on Metaprogramming:
http://docs.julialang.org/en/release-0.5/manual/metaprogramming/

One fact that it misses: How to interpolate an expression as an
expression (not as a value) into another expression.

    \subsubsection{Metaprogramming Project}\label{metaprogramming-project}

You have a way to compute least squares, how about letting it work like
an \texttt{R} function? Define your own version
\texttt{solve\_least\_squares(Y,X)} using your previous result, and
define the \texttt{@\textasciitilde{}} macro so that way
\texttt{solve\_least\_squares(Y\textasciitilde{}1+X1+X2+X4)} solves the
least square problem on the data matrix \texttt{X} on its columns 1, 2,
and 4.

Metaprogramming is hard the first few times you do it. You might want to
work in pairs!

Hints:

\begin{itemize}
\itemsep1pt\parskip0pt\parsep0pt
\item
  You cannot ``get a character'' from a symbol
\item
  You can use \texttt{string} to change a symbol to a string
\item
  You can \texttt{parse} a string to a number
\item
  Check out the \texttt{args} field of an expression: it's an array that
  holds a bunch of goodies!
\end{itemize}
